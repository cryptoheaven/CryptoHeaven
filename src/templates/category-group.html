Groups and Role Based Access Control

Using groups to control access to shared resources simplifies administrative burden when adding or removing members. With group access controls, simply assign a new member to a group and all previously set-up resources become available to the new member of the group.

Group Memberships

A user can be a member of more then one group. A group can contain any number of members, including other groups.


Empty Groups

Empty groups/roles are allowed. Create empty groups and use them to build your CryptoHeaven environment. You'll be able to add members later.
